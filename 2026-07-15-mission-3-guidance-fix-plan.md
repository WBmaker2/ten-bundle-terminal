# 3/9 미션 조작 안내 개선 계획

**작성일:** 2026-07-15  
**대상:** `처음 만드는 십 묶음` 미션과 공통 묶기·풀기 작업대

## 1. 재현된 문제

1. 3/9 미션에서 `낱개 10개 묶기`를 누르면 목표 모습인 `십 묶음 1개·낱개 0개`가 됩니다.
2. 학습자가 탐색 과정에서 다시 풀었다가 묶으면 최종 모습은 다시 정확해집니다.
3. 기존 판정은 전체 조작 기록이 필요한 순서와 길이까지 완전히 같아야 통과시킵니다.
4. 따라서 최종 모습과 마지막 조작이 모두 맞아도 `이번 배송에서 필요한 조작 순서를 차례로 해 보세요.`라는 추상적인 문장만 표시되어 진행할 수 없습니다.
5. 작업대 화면에는 미션 소개에서 보았던 목표가 사라지고, 현재 눌러야 할 버튼을 가리키는 안내도 없습니다.

## 2. 개선 원칙

- 탐색 과정의 실수를 벌점처럼 누적하지 않습니다.
- 필요한 조작 순서를 **마지막에 올바르게 수행했는지** 확인합니다.
- 작업대에서 `지금 할 일`을 버튼 이름과 동일한 말로 표시합니다.
- 순서가 두 단계인 미션은 `1 → 2` 진행 상태를 항상 보여 줍니다.
- 순서를 마치면 다음 행동인 `묶음 확인`을 명확히 안내합니다.
- 기존 버튼의 접근 가능한 이름과 학습 흐름은 유지합니다.

## 3. 구현 내용

### 조작 순서 판정

- 전체 기록의 완전 일치 대신, 조작 기록의 마지막 부분이 필요한 순서와 일치하는지 판정합니다.
- 예: `묶기 → 풀기 → 묶기`는 마지막 `묶기`가 3/9의 목표 순서이므로 통과합니다.
- 7/9의 `풀기 → 묶기`처럼 두 단계 학습 목표는 마지막 두 동작이 해당 순서일 때 통과합니다.

### 화면 안내

- 작업대에 `이번에 할 순서` 카드를 추가합니다.
- 아직 해야 할 동작은 `지금 할 일: 낱개 10개 묶기`처럼 표시합니다.
- 단계별 칩에 `할 차례`, `완료` 상태를 문자와 색으로 함께 표시합니다.
- 현재 필요한 동작 버튼을 강조합니다.
- 순서를 마치면 `순서 완료! 아래 묶음 확인을 눌러 주세요.`라고 표시합니다.

### 오류 문구

- 목표 모습이 다를 때는 다음에 눌러야 할 실제 버튼 이름을 알려 줍니다.
- 순서를 시작하지 않고 확인했을 때는 전체 순서를 번호와 버튼 이름으로 안내합니다.

## 4. 테스트

- 조작 기록의 마지막 부분이 필요한 순서와 일치하는 판정 단위 테스트를 추가합니다.
- 3/9에서 `묶기 → 풀기 → 묶기 → 묶음 확인`이 통과하는 브라우저 테스트를 추가합니다.
- 3/9 작업대에 `지금 할 일: 낱개 10개 묶기`가 보이는지 확인합니다.
- 7/9의 `풀기 → 묶기` 순서 요구가 유지되는지 기존 회귀 테스트로 확인합니다.
- 전체 단위 테스트, 접근성 검사, 반응형 검사와 빌드를 다시 실행합니다.

## 5. 업데이트와 배포

- 앱 업데이트 내역에 2026-07-15 조작 안내 개선을 추가합니다.
- 검증 후 커밋·푸시하고 기존 공개 Sites 주소에 새 버전을 배포합니다.
