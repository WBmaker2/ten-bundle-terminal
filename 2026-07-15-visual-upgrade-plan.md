# 십 묶음 택배 터미널 밝은 톤 이미지 고도화 계획

**작성일:** 2026-07-15  
**목표:** 기존 학습 흐름과 접근성 계약을 유지하면서, 초등 1~2학년 학생이 `낱개 10개 ↔ 십 묶음 1개` 관계를 더 빠르게 이해할 수 있도록 앱 전체를 밝고 친근한 택배 터미널 시각 언어로 개편합니다.

## 1. 현재 화면 진단

- 첫 화면의 학습 문장은 명확하지만, 큰 면적의 단색 카드와 어두운 브라우저 표현 때문에 저학년용 학습 앱보다 관리 화면처럼 보일 수 있습니다.
- 낱개와 십 묶음의 코드 기반 모형은 수량 정확성에 적합하므로 유지하되, 상자·테이프·선반의 시각 단서를 강화합니다.
- 생성 이미지는 장식과 맥락 이해에 사용하고, 정답 판정에 필요한 수량은 이미지 안에 맡기지 않습니다.
- Safari의 자동 어두운 표현을 피하도록 `color-scheme: light`와 명시적 배경·텍스트 색을 적용합니다.

## 2. 시각 방향

- **톤:** 햇빛이 드는 밝은 교실형 택배 터미널
- **주색:** 하늘색, 민트, 살구색, 노랑, 따뜻한 크림색
- **형태:** 큰 둥근 모서리, 흰 카드, 얕은 그림자, 굵고 명확한 테두리
- **정보 계층:** 화면당 핵심 문장 1개 → 조작 영역 → 현재 수량 관계 → 다음 행동 순서
- **저학년 친화:** 버튼 최소 48px, 짧은 라벨, 상태별 색상과 텍스트 병기, 과도한 장식 배제

## 3. 생성 이미지 계획

### 3.1 첫 화면 핵심 삽화

- 용도: 시작 화면의 택배 터미널 세계관과 `여러 낱개를 한 묶음으로 정리한다`는 활동 맥락 전달
- 형식: 밝은 3D 종이 공예풍 또는 부드러운 클레이풍의 가로형 삽화
- 구성: 작은 주황색 택배 상자들이 컨베이어를 따라 파란 묶음 상자로 정리되는 장면, 친근한 터미널 설비, 넉넉한 여백
- 금지: 글자, 숫자, 로고, 워터마크, 실제 정답으로 오해할 수 있는 복잡한 수량 배열
- 접근성: 장식 이미지로 처리하고, 핵심 개념은 바로 아래의 코드 기반 `낱개 10개 = 십 묶음 1개` 모형과 대체 텍스트로 제공합니다.

### 3.2 소셜 미리보기 이미지

- 용도: 링크 공유 카드
- 형식: 앱의 밝은 색상과 상자 모티프를 반영한 가로형 카드
- 필수 문구: `십 묶음 택배 터미널`, `10개를 묶고 풀며 수를 배워요`
- 생성 결과의 글자가 부정확하면 한 번만 재시도하고, 통과한 이미지에만 Open Graph 메타데이터를 연결합니다.

## 4. 화면별 개편

### 시작 화면

- 헤더와 본문을 밝은 크림 배경 위 흰 카드로 재구성합니다.
- 생성 삽화 아래에 정확한 코드 기반 개념 스트립을 배치합니다.
- `연습 시작` 버튼을 가장 강한 파란색 행동 요소로 유지합니다.
- `고정 9개 미션`, `점수 없음`, `학생 정보 저장 안 함`을 아이콘형 안내 칩으로 정돈합니다.

### 연습·미션·검수 화면

- 십 묶음 영역은 하늘색, 낱개 영역은 살구색으로 일관되게 구분합니다.
- 상자 모형에 테이프와 상자 면을 추가해 실제 택배 상자로 쉽게 인식되도록 합니다.
- 전체 수 관계는 어두운 막대 대신 밝은 파란색 요약 카드로 표시합니다.
- 조작 버튼은 `묶기`, `풀기`, `되돌리기`의 우선순위와 상태를 색·테두리·문장으로 함께 표현합니다.

### 완료·정리 화면

- 성공 상태는 민트색 카드와 작은 축하 장식으로 표현합니다.
- 읽기·숫자·자리값 카드는 같은 높이와 간격으로 정리합니다.
- 생각 정리 문항은 선택 전후가 색뿐 아니라 테두리와 체크 상태로도 구분되게 합니다.

## 6. 검증 기준

- 기존 단위·컴포넌트·E2E 테스트가 모두 통과합니다.
- 320px, 375px, 768px, 1280px에서 가로 넘침이 없습니다.
- 첫 화면의 심각·치명적 자동 접근성 위반이 없습니다.
- Safari를 포함한 브라우저에서 시스템 다크 모드와 무관하게 밝은 테마로 표시됩니다.
- 생성 이미지는 실제 프로젝트 파일로 저장되고 앱에서 불러옵니다.
- 모든 코드 파일은 500줄 미만을 유지합니다.
- 업데이트 내역에 2026-07-15 시각 개선 내용을 추가합니다.
