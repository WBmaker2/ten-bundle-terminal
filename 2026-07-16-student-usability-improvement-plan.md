# 초등학생 실사용 테스트 후 개선 구현 계획

**작성일:** 2026-07-16
**대상 버전:** v1.3.0 → v1.4.0
**승인 상태:** 사용자가 `구현 시작`을 요청하여 별도 계획 검토 대기 없이 진행

## 목표

초등학교 1~2학년 사용자의 실제 학습 흐름에서 발견한 수학적 정합성, 작은 화면, 진행 안내, 문구 난이도 문제를 고치고 완료 후 선택 활동의 부담을 줄입니다.

## 구현 범위

1. `2묶음 10낱개`처럼 계산 결과가 30인 표현은 맞는 수량으로 인정하고, 가장 간단한 묶음 모습도 함께 안내합니다.
2. 남은 미션 수에 현재 미션을 포함하여 마지막 미션 전에는 `1개 남음`으로 표시합니다.
3. 42 문항을 `십 묶음 4개와 낱개 2개`를 직접 보도록 쉬운 문장으로 바꿉니다.
4. 완료 화면의 생각 정리 3문제를 기본 접힘 선택 활동으로 바꿉니다.
5. 320px 수량 입력 화면의 가로 넘침을 제거하고 모바일 상단 버튼 높이를 보완합니다.
6. 앱 버전과 업데이트 내역을 v1.4.0으로 갱신합니다.

## 완료 조건

- 30 보너스에서 `3묶음 0낱개`와 `2묶음 10낱개`가 모두 30임을 올바르게 설명합니다.
- 9번째 미션을 시작하기 전 `1개 남음` 또는 동일 의미가 표시됩니다.
- 42 수량 문항이 저학년에게 구체적인 문장으로 표시됩니다.
- 완료 직후 생각 정리 문제는 접혀 있고 학생이 원할 때 펼칠 수 있습니다.
- 320×568 수량 입력 화면에서 `scrollWidth <= clientWidth`입니다.
- 모바일 상단의 작은 버튼 높이가 최소 44px입니다.
- 기존 9개 미션, 복습, 업데이트 내역, 접근성 흐름이 회귀하지 않습니다.

## 검증 계획

- 변경 로직의 단위·컴포넌트 테스트 추가
- Chromium E2E에 30 대체 표현, 마지막 진행 안내, 42 문구, 접힌 생각 정리, 320px 수량 입력 화면 회귀 테스트 추가
- `npm test`, `npm run lint`, `npm run typecheck`, `npm run build`
- `npm run check:lines`, `npm run check:content`, `git diff --check`
- 공개 또는 로컬 화면에서 320px와 일반 화면 직접 클릭 재검증

## 제약과 위험

- 전체 택배 수는 묶기·풀기 전후에 변하지 않아야 합니다.
- 낱개 10개 이상 표현도 수량상 맞을 수 있지만, 표준 묶음 모습과는 구분해 설명합니다.
- 모든 코드 파일은 500줄 미만을 유지합니다.
- 밝은 저학년용 디자인과 기존 업데이트 내역 버튼을 유지합니다.

## 작업 기록

- 계획 작성: 실사용 테스트에서 확인한 5개 개선 항목을 v1.4.0 범위로 확정했습니다.
- 구현 기록: 같은 수를 나타내는 보너스 답 판정, 남은 미션 계산, 42 문구, 접힌 생각 정리, 320px 그리드와 모바일 버튼 높이, v1.4.0 업데이트 내역을 반영했습니다.
- 구현 중 추가 수정: 목표 E2E에서 좁은 요약 카드의 택배 그림이 질문 카드 위로 겹치는 문제를 발견하여 내부 묶음·낱개 그리드도 축소 가능하도록 고쳤습니다. 작은 화면 문제와 같은 원인의 범위 내 조정입니다.
- 검증 기록: 단위·컴포넌트 85개, Chromium E2E 28개, lint, typecheck, build, 파일 길이, 콘텐츠 정책, diff 공백 검사를 모두 통과했습니다.
- 실제 브라우저 기록: 로컬 운영 빌드의 320px 화면에서 `scrollWidth 320px`, 숫자 키패드 입력값 `0`, 데스크톱 수 읽기 선택 완료, 콘솔 오류 0건을 확인했습니다.
- 계획 대비 리뷰: 계획한 완료 조건을 모두 충족했으며 미완료 항목이나 의도하지 않은 동작 변경은 없습니다.
