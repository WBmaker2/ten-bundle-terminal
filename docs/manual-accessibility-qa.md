# 십 묶음 택배 터미널 수동 접근성 검수표

**기준 버전:** v1.3.0
**작성일:** 2026-07-15
**상태:** 실제 기기 검수 대기

자동 테스트로 확인하기 어려운 항목을 실제 브라우저와 기기에서 기록하기 위한 문서입니다. 검수할 때 날짜, 운영체제, 브라우저 버전, 기기를 함께 적습니다.

## 1. macOS VoiceOver와 Safari

- [ ] 첫 화면에서 `십 묶음 택배 터미널` 제목과 학습 목표가 자연스러운 순서로 낭독됩니다.
- [ ] 화면 전환 뒤 새 제목으로 초점이 이동합니다.
- [ ] 진행 표시가 `완료한 수, 현재 미션, 남은 수` 순서로 한 번만 낭독됩니다.
- [ ] 십 묶음 수, 낱개 수, 전체 수가 상자 그림을 보지 않아도 이해됩니다.
- [ ] `지금 할 일`, 조작 성공, 오류, 순서 완료 안내가 중복 낭독되지 않습니다.
- [ ] 업데이트·교사용 안내·초기화 대화상자를 열면 초점이 안으로 이동합니다.
- [ ] 대화상자를 닫으면 원래 누른 버튼으로 초점이 돌아옵니다.
- [ ] 복습과 보너스 연습을 키보드와 VoiceOver만으로 완료할 수 있습니다.

## 2. 실제 터치 기기

- [ ] 묶기, 풀기, 되돌리기, 수량 확인 버튼이 한 번의 탭으로 작동합니다.
- [ ] 빠르게 연속 탭해도 수량이 잘못 바뀌거나 화면이 흔들리지 않습니다.
- [ ] 숫자 키패드와 선택 카드의 터치 범위가 충분합니다.
- [ ] 세로 스크롤 중 버튼이 의도치 않게 눌리지 않습니다.
- [ ] 320~375px 폭에서 진행 표시와 헤더 버튼이 겹치지 않습니다.
- [ ] `현재 택배 모습`을 접고 펼칠 수 있으며 핵심 조작 버튼은 쉽게 찾을 수 있습니다.

## 3. 브라우저 200% 확대

- [ ] Safari 200%에서 시작 화면의 제목, 삽화, 연습 시작 버튼이 잘리지 않습니다.
- [ ] Chrome 200%에서 헤더가 두 줄 이내로 정리되고 버튼이 겹치지 않습니다.
- [ ] 작업대에서 `지금 할 일`과 해당 조작 버튼을 같은 화면 구간에서 찾을 수 있습니다.
- [ ] 수량 입력과 화면 키패드가 가로로 넘치지 않습니다.
- [ ] 완료 카드와 배송 정리표 문장이 카드 밖으로 잘리지 않습니다.
- [ ] 대화상자 내부를 스크롤해 닫기와 확인 버튼에 도달할 수 있습니다.
- [ ] sticky 또는 고정 요소가 본문과 버튼을 가리지 않습니다.
